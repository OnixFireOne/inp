# DESIGN.md — inp.one v2

> Дизайн-спецификация для ИИ-разработчика. Цель: лёгкий, спокойный, современный по стандартам 2025 интерфейс. Структура экранов прежняя (таблица → drawer), но **вид новый** — старый Angular-стиль НЕ переносим. Код/идентификаторы — на английском, пояснения — на русском.

---

## 0. Зафиксированные решения (короткий список)

- **Акцент:** индиго / сине-фиолетовый, один токен `--accent` (легко поменять позже).
- **Темы:** тёмная по умолчанию + светлая переключателем.
- **Плотность строк:** комфортная (~56px) по умолчанию + переключатель «Компактно».
- **Карточка актива:** на desktop — **боковой drawer справа, немодальный**; на mobile — **bottom-sheet (vaul)**.
- **Шапка карточки:** sticky (кнопка закрытия и вкладки не скроллятся).
- **Раскладка ссылок:** заголовки-категории сохраняем; внутри — гибрид «список (Core) + чипы (Trusted)», иконка + название (не голые иконки).
- **Метрики в «Обзоре»:** на виду только Цена / 24ч% / Капитализация / Объём; High/Low, ATH/ATL, Supply — под «Подробнее».
- **Спарклайн в таблице:** 24ч, свой инлайн-SVG, цвет по знаку 24ч.

---

## 1. Три принципа

1. **Лёгкость** — воздух вместо линий. Убираем тяжёлые границы и рекламный баннер сверху. Контент дышит за счёт отступов, а не рамок.
2. **Спокойствие** — приглушённая палитра. Цвет (зелёный/красный) только на изменении цены. Остальное нейтральное.
3. **Ясность** — с первого взгляда понятно, что кликабельно: имя и график подсвечиваются, курсор-указатель + иконка действия.

---

## 2. Что меняем относительно текущего сайта

| Сейчас (старый сайт) | Стало (v2) |
|---|---|
| Рекламный баннер Binance сверху давит на контент | Убираем сверху; промо — деликатным блоком сбоку/внизу или нет |
| Плотные ряды, мелкий текст, всё близко | Высота строки ~56px, крупнее имя, воздух между колонок |
| Не видно, что имя/график кликабельны | Явные hover-состояния + курсор + иконка «открыть» |
| Колонка «7d» спарклайн | Спарклайн **24ч**, свой SVG, цвет по 24ч |
| Жёсткие линии сетки | Borderless: только тонкий разделитель строк, без вертикальных линий |
| Детали в иконках-плашках без подписей | Иконка + название, иерархия Core → Trusted |

## 3. Цвет и тема (дизайн-токены)

- Тёмная тема по умолчанию, фон не чёрный, а мягкий тёмно-серо-синий.
- Нейтральная база: 4–5 оттенков серого. Семантика (зелёный/красный) — только на цене, приглушённая.
- Один акцент (`--accent`). Больше одного акцента = шум.

```css
/* Дизайн-токены (CSS vars). Тёмная тема по умолчанию. */
:root[data-theme="dark"] {
  --bg:        #0e1116;   /* фон страницы — мягкий, не чёрный */
  --surface:   #161a22;   /* карточки, drawer, строки при hover */
  --surface-2: #1d222c;   /* вложенные поверхности */
  --border:    #232936;   /* тончайшие разделители */
  --text:      #e6e9ef;   /* основной текст */
  --text-mut:  #8b93a3;   /* тикеры, подписи, второстепенное */
  --accent:    #6e8bff;   /* ссылки, активная вкладка, focus ring */
  --up:        #3ec98a;   /* рост (приглушённый зелёный) */
  --down:      #f0666b;   /* падение (приглушённый красный) */
  --radius:    14px;      /* мягкие скругления */
}
:root[data-theme="light"] {
  --bg:#fafbfc; --surface:#ffffff; --surface-2:#f3f5f8;
  --border:#e7eaf0; --text:#10141b; --text-mut:#5b6472;
  --accent:#4f6bff; --up:#16a36b; --down:#e0454b; --radius:14px;
}
```

Переключатель темы хранит выбор (`localStorage`), уважает `prefers-color-scheme` при первом заходе.

## 4. Типографика и числа

- Шрифт — **Inter** (или системный стек).
- **Цены и проценты — табличные цифры** (`font-variant-numeric: tabular-nums`), чтобы колонки не «прыгали» при обновлении котировок.
- Иерархия: имя актива 15px/600, тикер 13px/500 muted, цена 15px/500 tabular, проценты 14px.

---

## 5. Таблица: анатомия строки

```
#   [icon] Имя ↗        Цена        Кап.       24ч %      [▁▂▃▅▇ спарклайн] ⤢
           TICKER
```

- **Колонки:** № · Название (icon + имя + тикер) · Цена · Капитализация · 24ч % · Спарклайн 24ч.
- **Высота ~56px** (комфортная плотность по умолчанию), без вертикальных линий — только тончайший `--border` снизу. Переключатель «Компактно» уменьшает высоту строки.
- **Состояния строки:**
  - *Default* — прозрачный фон.
  - *Hover* — фон `--surface` (плавно 150ms), курсор-указатель, в конце строки проявляется шеврон `›`.
  - *Focus-visible* — кольцо `--accent`.
  - *Active/press* — лёгкое затемнение.

## 6. Кликабельность: имя и график (КЛЮЧЕВОЕ)

Два разных действия по клику, оба читаются без обучения:

| Элемент | Что делает клик | Как показываем кликабельность |
|---|---|---|
| **Название валюты** | Открывает карточку, вкладка «Обзор» | `<a href="/asset/[id]">`: hover → цвет `--accent` + подчёркивание, курсор-указатель, рядом иконка `↗`. Полноценная ссылка (ПКМ «открыть в новой вкладке»). |
| **Спарклайн (график)** | Открывает карточку сразу на вкладке «График» | `<button>`: hover → подложка `--surface-2`, scale 1.02, иконка `⤢`, тултип «Открыть график». |
| Остальная строка | Открывает карточку (вкладка «Обзор») | Вся строка — большая зона клика; hover-фон + шеврон `›`. |

**Принцип affordance:** кликабельное всегда даёт 3 сигнала одновременно — (1) курсор-указатель, (2) смена вида при наведении, (3) иконка-намёк (`↗` / `⤢` / `›`).

**Не вложенные клики:** спарклайн-кнопка перехватывает событие (`stopPropagation`) и ведёт на «График», а не «Обзор».

---

## 7. Шапка страницы и фильтры (вместо баннера)

- Лёгкая панель сверху: логотип + **поиск** (широкий, скруглённый) + сегменты **Все / Watchlist / В тренде** + переключатель темы.
- Никакого рекламного баннера в потоке контента.
- Сортировка — клик по заголовку колонки (стрелка ↑/↓).

---

## 8. Карточка актива (drawer справа / sheet снизу)

- **Desktop — боковой drawer справа** (side sheet), ~420–460px. **Немодальный**: без перекрывающего затемнения, таблица слева остаётся **кликабельной**.
  - Почему drawer, а не центрированный модал: контент — длинный каталог ссылок по категориям; боковая панель скроллится и не «накрывает» весь экран; сохраняется контекст списка (клик по другой строке = смена актива). Так делают Linear, Stripe, Vercel.
- **Mobile — bottom-sheet (vaul)**: тянется снизу, «ручка» сверху, свайп вниз закрывает (модальный режим — таблица скрыта).
- Технически — один компонент `ResponsiveSheet`: Radix Dialog с боковым позиционированием на desktop, vaul на mobile. Deep-link `inp.one/asset/[id]` (intercepting route) одинаково открывает обе формы.
- Внутри: sticky-шапка → вкладки **Обзор / График** → контент.

### 8.1. Поведение клика вне drawer

- **Desktop (немодальный):** клик по таблице слева **не закрывает** drawer — он **переключает** панель на другой актив. Закрытие — крестик `×` или Esc.
- **Mobile (модальный):** закрытие свайпом вниз / крестиком / Esc / тапом по затемнённому фону.
- Причина: затемнение, закрывающее по клику, мешало бы выбирать валюты из списка.

### 8.2. Sticky-шапка карточки и кнопка закрытия

- Верхний блок `position: sticky; top: 0` внутри скролл-контейнера drawer. Шапка (иконка + имя + цена + 24ч + крестик + вкладки) всегда на экране.
- Раскладка (две строки, компактно):

```
[icon] Bitcoin (BTC)              [☆ watchlist]  [×]
$64 015   +3.47% (24ч)
[ Обзор | График ]   ← вкладки тоже sticky, под шапкой
```

- Кнопка `×`: зона клика ≥ 40×40px, `aria-label="Закрыть"`, видимый `focus-visible`.
- Закрыть: крестик · Esc · (desktop) только эти; (mobile) ещё свайп вниз и тап по фону.
- При скролле под шапкой появляется тонкая линия `--border` / лёгкая тень.
- Шапка компактная: две строки + вкладки, остальное место — контенту.

## 9. Вкладка «Обзор»: метрики + ссылки

### 9.1. Блок метрик (что показывать)

Это каталог ссылок, а не торговый терминал — глубокие данные даёт «График» (TradingView). Поэтому в «Обзоре» — лёгкая полоса контекста, два уровня:

**Всегда видно (главное):**
```
Цена $64 015     24ч +3.47%     Кап. $1.28T     Объём 24ч $38B
```

**Под «Подробнее» (свёрнуто по умолчанию):**
```
High/Low 24ч $65 100 / $62 800     ATH/ATL (за всё время)     Supply 19.7M / 21M
```

Рекомендация по приоритетам:
- **Цена, 24ч %, Капитализация, Объём 24ч — всегда.**
- **High/Low 24ч — во второй уровень** («Подробнее»).
- **ATH/ATL («за всё время») — во второй уровень, по желанию.**
- **Supply (циркулирующий/макс.) — нишевая, под «Подробнее»** (или вовсе не показывать).
- Источник — тот же CoinGecko (`high_24h`, `low_24h`, `total_volume`, `market_cap`, `ath`, `atl`, `circulating_supply`, `max_supply`). Если метрики нет — строка скрывается, без пустых прочерков.
- Числа — `tabular-nums`, рост/падение — семантическим цветом.

### 9.2. Раскладка ссылок по категориям

Старый сайт: заголовок категории → сетка иконок-плашек 56×56 без читаемых подписей + GIF-превью. **Вердикт: паттерн ок, но реализация устарела** (опора на узнавание, нет иерархии, тяжёлые GIF, плотная сетка мелких целей).

**Раскладка v2 — гибрид «список + чипы»** (заголовки-категории сохраняем):
- **Core-ссылки (топ 2–4 в категории) — богатые строки:** `[иконка] Название — короткое описание ↗`. Подпись видна всегда.
- **Остальные (Trusted) — компактные чипы:** пилюля `[иконка] Название` (иконка + текст рядом, не голая иконка).
- **Иконки — статичные фавиконы/логотипы** (без GIF), чёткие.
- **Категории — сворачиваемые секции** (accordion): важные (Charts, Aggregators) раскрыты, редкие (Team, Token unlocks) свёрнуты.

### 9.3. Макро-меню по клику на ссылку

У части источников в данных одна плитка содержит несколько вариантов перехода (поле `urls` у ссылки в `catalog.seed.json`).
- Один адрес → клик открывает сразу в новой вкладке (`target="_blank" rel="noopener"`).
- Несколько → клик открывает **поповер-меню** (Radix Popover / DropdownMenu) рядом с плиткой; на плитке деликатный значок `▾`.
- Меню закрывается по клику вне / Esc, навигация стрелками (доступность).
- Каждый переход шлёт `link_click` в `/api/events` (для ИИ-ранжирования) — и для одиночных, и для выбора из меню.

## 10. Вкладка «График»

- Виджет TradingView (Advanced), грузится лениво, но скрипт `tv.js` прогрет в фоне → открывается мгновенно.
- `tvSymbol`, например `BINANCE:BTCUSDT`. Переключение монеты — через `setSymbol`, без пересоздания виджета.

---

## 11. Спарклайн в таблице

- 24ч, свой инлайн-SVG (~24 точки), цвет по знаку 24ч (`--up`/`--down`).
- Данные — CoinGecko `market_chart?days=1` через `/api/sparklines` (Redis-кэш ~5 мин).
- Это `<button>` → клик открывает карточку на вкладке «График».

---

## 12. Микро-взаимодействия

- Переходы — 150–200ms ease-out, без резких скачков.
- Появление цен — fade-in из скелетона, без сдвига разметки (CLS≈0) за счёт `tabular-nums` и фиксированной ширины колонок.
- Watchlist-звезда — optimistic, мгновенно с лёгким «pop».
- Hover по строке — плавная подложка.
- Уважение `prefers-reduced-motion`.

---

## 13. Доступность (по стандартам)

- Контраст текста ≥ WCAG AA.
- Полная навигация с клавиатуры: строки и кнопки фокусируемы, видимое `focus-visible` кольцо.
- Имя — настоящая ссылка (`<a>`), график — `<button aria-label="Открыть график BTC">`.
- Цвет не единственный носитель смысла: у % изменения — знак `+/−` и стрелка (для дальтоников).

---

## 14. Бюджеты производительности

- LCP < 1.5s, CLS < 0.05, INP < 200ms.
- Открытие drawer < 100ms.
- Первый экран без обращения к БД (ISR/статика каталога); цены грузятся отдельно со скелетонами.
- TradingView — динамический импорт, без layout shift.
